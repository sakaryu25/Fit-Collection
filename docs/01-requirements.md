# 01. 要件定義

## 1. プロダクト概要

| 項目 | 内容 |
|---|---|
| アプリ名 | Fit Collection（仮） |
| 一言コンセプト | 買う前に、自分のクローゼットと合うか確認できる、部屋付きアバター着せ替えアプリ |
| ターゲット | 10代後半〜20代のファッション好き。ECや店頭で「これ手持ちと合うかな」と迷う人 |
| プラットフォーム | iOS / Android（React Native / Expo） |
| 課金 | MVPでは無料。将来的にサブスク（AI機能・部屋カスタマイズ）を検討 |

## 2. 解決する課題

ユーザーが服を買うとき・買おうか迷うときの悩み：

1. これ欲しいけど、自分の持ってる服と合うかな
2. 買っても着回しできるかな
3. 自分っぽいコーデになるかな / 自分の系統と合うかな

→ **自分のアバターに「手持ち服 + 迷っている服」を組み合わせて着せて確認できる**ことで解決する。

## 3. プロダクト原則（判断に迷ったらここに戻る）

1. **人物はデフォルメ、服は忠実** — アバターは可愛く親しみやすく。服は色・柄・シルエット・丈感が「その服っぽい」こと。
2. **部屋が拠点** — アプリを開いたら自分の部屋。クローゼットは部屋の中にある世界観の一部。
3. **管理アプリではなく体験アプリ** — 一覧画面の効率より、着せ替えの楽しさを優先しつつ、実用性を損なわない。
4. **MVPは「体験の成立」優先** — 完全再現より、ループ（登録→着せ替え→判断）が回ることが先。

## 4. ユーザーストーリー

### コアループ

```
アプリを開く → 自分の部屋にアバターがいる → クローゼットを開く
→ 手持ち服を見る → 着せ替える → 気になる服を追加する → 仮試着する
→ 手持ち服と組み合わせる → コーデを保存する → 買うかどうか判断する
```

## 5. 機能要件

### F-01 アバター作成

- パーツ選択方式のアバタークリエイト（性別・雰囲気 / 肌色 / 顔型 / 目 / 眉 / 口 / 髪型 / 髪色 / 体型 / 身長イメージ）
- リアル再現ではなく「自分っぽい雰囲気」が伝わればよい
- 作成後いつでも編集可能
- 初回起動時のオンボーディングで作成（スキップ不可。アバターが体験の中心のため）

### F-02 手持ち服の登録（クローゼットアイテム）

- 登録方法: カメラ撮影 / 写真フォルダから選択
- 登録項目: 画像（必須）/ カテゴリ（必須）/ 色（必須・パレット選択）/ 系統タグ / 季節 / ブランド名 / アイテム名 / メモ
- カテゴリ: トップス / ボトムス / アウター / ワンピース / シューズ / バッグ / アクセサリー / 帽子
- 系統: カジュアル / ストリート / モード / きれいめ / 古着 / 韓国系 / Y2K / シンプル（複数選択可）
- 登録時に「アバター表示用の見え方」（テンプレート形状 + 色/柄）を設定する（詳細は 03 ドキュメント）

### F-03 クローゼット

- カテゴリ別 / 色別 / 季節別 / ブランド別の表示・絞り込み
- お気に入り、検索（名前・ブランド・メモ）、並び替え（追加日 / 色 / カテゴリ）
- アイテム詳細（画像・メタ情報・このアイテムを使ったコーデ一覧・「着せてみる」ボタン）
- 入口は2つ: **部屋の中のクローゼット（世界観）** と **タブからの一覧画面（実用）**。中身は同じデータ。

### F-04 着せ替え

- スロット方式: トップス / ボトムス / アウター / ワンピース / シューズ / バッグ / アクセ / 帽子
- ワンピースはトップス+ボトムスと排他
- カテゴリタブで服を切り替えながらリアルタイムにアバターへ反映
- アバターの回転（ドラッグ）、将来的にズーム・ポーズ変更
- 「保存」でコーデ化、「リセット」「ランダム」補助ボタン

### F-05 気になる服（ウィッシュリスト）の登録

- 登録元: ECスクショ / 商品画像 / 店頭写真
- 項目: 画像（必須）/ カテゴリ（必須）/ 色 / ブランド名 / 商品名 / 値段 / URL / メモ
- ステータス: 欲しい / 保留 / 買った / いらない
- 「買った」にするとクローゼットへの移動を提案（データはコピーし、ウィッシュリスト側に履歴を残す）

### F-06 仮試着

- 気になる服を着せ替え画面で手持ち服と同様に着せられる（見た目上「検討中」バッジで区別）
- 仮試着の履歴を記録（try_on_history）
- 仮試着から直接コーデ保存できる（コーデに検討中アイテムが含まれることを許容）

### F-07 相性確認（MVPは ルールベース、将来AI）

MVPでの簡易ロジック（クライアント側で計算）:
- **色相性**: 登録色同士の組み合わせ表（モノトーン×全色=◎、暖色×寒色=△ など）
- **系統相性**: 系統タグの一致・隣接度（カジュアル↔ストリート=近い、モード↔Y2K=遠い など）
- **季節整合**: 季節タグの一致
- 出力: 相性スコア（0-100）/ 合わせやすい手持ちアイテムTOP5 / おすすめコーデ例（トップス+ボトムス+シューズの自動組成）/ 着回ししやすさ（合わせられる手持ち服の数）/ 買うべき度

表示例:
> このジャケットは、あなたの黒スラックス、白Tシャツ、黒ローファーと相性が良いです。モード・きれいめ寄りのコーデに使いやすく、着回ししやすいアイテムです。

### F-08 コーデ保存

- 保存内容: コーデサムネイル画像（3Dビューのスナップショット）/ 使用アイテム一覧 / コーデ名 / シーン / 季節 / メモ
- シーン: 学校 / デート / カフェ / 遊び / 旅行 / バイト / イベント / 飲み会
- コーデ一覧から「このコーデを着せる」で再現可能

### F-09 マイルーム

- アプリのホーム = 部屋ビュー。アバターが部屋に立っている
- 部屋内のクローゼットをタップ → クローゼットが開く
- 部屋内のミラー（鏡）をタップ → 着せ替えモードへ
- MVPの部屋は固定レイアウト1種（白・グレー・ベージュ基調のシンプルおしゃれな部屋）
- 将来: 家具配置・テーマ変更・クローゼット拡張

## 6. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | 部屋+アバター表示で実機30fps以上。ミドルレンジ端末（iPhone SE3 / Pixel 6a相当）で動作 |
| 起動 | コールドスタート〜部屋表示まで3秒以内（3Dアセットはローカルバンドル + キャッシュ） |
| オフライン | 閲覧・着せ替えはオフライン可（ローカルキャッシュ）。登録はオンライン必須でよい |
| 画像 | アップロード時にリサイズ（長辺1280px）・圧縮。サムネイル自動生成 |
| セキュリティ | Supabase RLSで全テーブル本人のみアクセス。Storageもユーザーフォルダ単位で制限 |
| プライバシー | 服画像・アバターは本人のみ閲覧（MVPでは共有機能なし） |
| 拡張性 | 服の見た目表現を抽象化し、将来のAI処理（背景透過・自動分類・3D化）を差し替え可能にする |

## 7. スコープ外（MVPでやらないこと）

- 写真からの服の完全自動3D化 / AIによる本格試着合成
- 背景透過・カテゴリ自動判定・色自動抽出（手動入力で代替。設計上は差し替え可能に）
- SNS共有・フォロー・投稿
- 部屋カスタマイズ
- AIコーデ提案（相性確認はルールベースで提供）
- 複数アバター
