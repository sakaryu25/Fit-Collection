# 03. アバター・マイルーム・着せ替え仕様

## 1. デフォルメ3Dアバター仕様

### 1.1 方向性

- トモダチコレクション風: リアルすぎず、丸みがあり、シンプルで見やすい
- **頭身: 約3.5〜4頭身**。低すぎる（2頭身）と服のシルエット・丈感が表現できず、高すぎる（6頭身〜）とリアル寄りになる。「服が映える最低限の胴・脚の長さ」を確保する
- スタイル: トゥーンシェーディング（輪郭線なし or 極薄）、ベイクした柔らかい陰影、マットな質感

### 1.2 パーツ構成（パーツ切り替え方式）

1体のベースボディ（リグ入りGLB）に対し、パーツをアタッチ/モーフで切り替える。

| パーツ | 実装方式 | MVPバリエーション数 |
|---|---|---|
| 体型 | ベースメッシュのブレンドシェイプ（slim / normal / wide の morph 2軸） | 3段階 |
| 身長 | ルートボーンのスケール（0.95 / 1.0 / 1.05） | 3段階 |
| 肌色 | ボディマテリアルの色差し替え | 6色 |
| 顔型 | ブレンドシェイプ（丸/卵/シャープ） | 3種 |
| 目 | **テクスチャ差し替え**（顔はフラットなテクスチャ顔にする） | 8種 |
| 眉 | テクスチャ差し替え | 5種 |
| 口 | テクスチャ差し替え | 5種 |
| 髪型 | メッシュ差し替え（headボーンにアタッチ） | 10種（ショート/ボブ/ミディアム/ロング/パーマ/坊主 等） |
| 髪色 | 髪マテリアルの色差し替え | 10色 |
| 雰囲気 | プリセット（上記の組み合わせテンプレ。フェミニン/ボーイッシュ/中性 等） | 6種 |

> 顔パーツをテクスチャ方式にするのが重要。3Dメッシュの目鼻口は工数が爆発する上、デフォルメ感が出しにくい。トモコレ・どうぶつの森系は実質フラット顔。
> 顔テクスチャは「目・眉・口のレイヤーをcanvasで合成 → 1枚のテクスチャに焼く」ことでドローコールも減る。

### 1.3 ボーン構成

標準ヒューマノイドの簡略版（hips / spine / chest / neck / head / 肩肘手 / 股膝足、計約20ボーン）。
服メッシュは同じスケルトンにスキニングして**ボーンを共有**する（着せ替えの基本手法）。

### 1.4 モーション（MVP）

- idle（部屋での待機: ゆっくり呼吸 + たまに首かしげ）
- react（タップ時: 手を振る）
- tryon_pose（着せ替え時: 棒立ちより少しポーズ）
- 将来: 歩く、座る、ポーズ集

## 2. 服の3D表現仕様 — 「テンプレ形状 × 動的テクスチャ」方式

### 2.1 基本アプローチ

写真からの自動3D化はMVPではやらない。代わりに:

```
服の見た目 = ① テンプレートメッシュ（形状・シルエット・丈感）
           × ② パラメトリックなテクスチャ（色・柄）
           × ③ オプションの写真転写（プリント/ロゴ部分）
```

ユーザーは登録時に「形」を選び、「色・柄」を設定する（写真から色は自動提案）。
これで「完全再現」ではなく「その服っぽい」を実現する。

### 2.2 テンプレートメッシュ一覧（MVP: 約30種）

| カテゴリ | テンプレ（シルエット・丈感のバリエーション） |
|---|---|
| トップス | 半袖T(ジャスト/オーバー) / 長袖T / シャツ / パーカー / ニット / タンクトップ |
| ボトムス | スキニー / ストレート / ワイド / ショート / スカート(ミニ/ロング) / プリーツスカート |
| アウター | デニムJK / レザーJK / ロングコート / ダウン / カーディガン / ブルゾン |
| ワンピース | ミニ / ロング / シャツワンピ |
| シューズ | スニーカー(ローテク/ボリューム) / ローファー / ブーツ / サンダル |
| バッグ | トート / ショルダー / リュック / ミニバッグ |
| アクセ・帽子 | キャップ / ニット帽 / ネックレス / メガネ |

各テンプレに調整パラメータ:
- **丈** (ブレンドシェイプ: -1〜+1。クロップド↔ロング)
- **ゆとり/オーバーサイズ感** (ブレンドシェイプ)
- **袖丈** (トップス系)
- **首元** (クルー/V/ハイネック: テクスチャ+小モーフ)

→ 「シルエット・丈感・袖の長さ・首元・オーバーサイズ感」の要件をテンプレ選択+2〜3スライダーでカバーする。

### 2.3 テクスチャ生成（色・柄・素材感）

- 各テンプレは**共通UVレイアウト**を持ち、柄テクスチャを差し替え可能
- 柄: 無地 / ボーダー / ストライプ / チェック / 花柄 / 迷彩 / カスタム（写真の一部を切り取ってタイル or 胸プリントとして転写）
- 色: 写真から自動抽出した主要色を提案（MVPは画像の縮小→ドミナントカラー抽出のみ。クライアントで可能）
- 素材感: マテリアルのプリセット（コットン=マット / デニム=ノイズ法線 / レザー=軽い光沢 / ニット=編みパターン）
- 実装: `react-native-skia` または offscreen canvas で柄テクスチャを合成 → `THREE.CanvasTexture` 相当としてGPUへ

### 2.4 着せ替えの仕組み

- 服メッシュはアバターと同一スケルトンにスキニング済み → 装着 = 「服メッシュをシーンに追加して同じskeletonをbind」
- 体型ブレンドシェイプは服側にも同名morphを用意し、ボディと同期して変形（貫通防止）
- アウターを着たらインナーの貫通が課題 → MVPは**「アウター装着時はトップスの袖・裾を縮めるmorph」+ ボディ非表示マスク**で簡易対応
- スロット: `tops / bottoms / outer / onepiece / shoes / bag / accessory / hat`（onepiece は tops+bottoms と排他）

### 2.5 アセット制作パイプライン

1. Blenderでベースボディ + 服テンプレを制作（共通スケルトン・命名規約 `tpl_tops_tshirt_over` 等）
2. glTF (GLB) でエクスポート。Draco圧縮 + テクスチャはKTX2/512px
3. アプリにバンドル（テンプレは全ユーザー共通なのでアプリ内蔵。追加分はSupabase Storageから配信+キャッシュ）
4. 1テンプレあたり目標: 3,000〜8,000ポリゴン、全装着時のシーン合計 < 60k ポリゴン

## 3. マイルーム仕様

### 3.1 構成（MVP）

- 固定レイアウトの1ルーム。8畳ワンルーム風。白・グレー・ベージュ・黒基調
- カメラ: 斜め俯瞰の固定カメラ（FOV狭め=35°でジオラマ感）。デバイスの傾きでわずかに視差（±2°）
- オブジェクト:

| オブジェクト | タップ時の挙動 |
|---|---|
| クローゼット（大、画面左） | 扉が開くアニメ → クローゼット一覧へ |
| ミラー（全身鏡） | カメラが寄る → 着せ替えモードへ |
| アバター | リアクション。長押しでアバター編集 |
| 服ラック | 「最近登録した服」ポップアップ（MVPはクローゼットと同遷移でも可） |
| ベッド/ラグ/観葉植物/小物 | 装飾（タップでぷるんと揺れるだけ。遊び心） |

- ライティング: ベイク済みライトマップ + 環境光のみ（リアルタイムシャドウなし。軽量化）
- 部屋全体で目標 < 30k ポリゴン、テクスチャはアトラス1〜2枚

### 3.2 演出（世界観をつくる小ネタ）

- 服を登録完了すると、部屋のクローゼットへ服アイコンが吸い込まれるアニメ
- コーデ保存時、ミラーの前でアバターがポーズ → カシャッとスナップ演出
- 時間帯で窓の外の色が変わる（朝/昼/夕/夜の4パターン。環境光色を変えるだけ）

### 3.3 将来拡張

- rooms テーブルに `layout_id` / `theme_id` / `furniture`(jsonb) を持たせておき、家具配置・テーマ変更に対応できるようにする（MVPではデフォルト値固定）

## 4. 仮試着・相性確認仕様

### 4.1 仮試着フロー

1. 欲しい服詳細 →「仮試着する」
2. wishlist_item にも closet_item と同じ「見え方設定」(テンプレ+色柄)を持たせてあるので、同じ仕組みで装着
3. 着せ替え画面では検討中アイテムに「検討中」バッジ
4. 試着するたびに try_on_history に記録（試着回数が「気になり度」の指標になる）

### 4.2 相性スコア（MVP ルールベース）

```
score = 0.45 * colorScore + 0.35 * styleScore + 0.20 * seasonScore

colorScore : 色相性マトリクス（12色×12色の静的テーブル）で
             手持ち服との平均相性。モノトーン・ベージュは万能色として高め
styleScore : 系統タグの距離。同一=1.0 / 隣接=0.7 / 遠い=0.2
             （隣接定義: カジュアル-ストリート-Y2K / モード-きれいめ / 古着-カジュアル / 韓国系-きれいめ 等）
seasonScore: 季節タグ一致率
```

- 出力に使う文章はテンプレ文生成（「{系統}寄りのコーデに使いやすい」「{色}の{カテゴリ}と好相性」）
- **着回ししやすさ** = score>=60 の手持ち服数を5段階化
- **買うべき度** = score と着回し数の合成 + 試着回数ボーナス
- 将来: この計算を Supabase Edge Function 経由の AI 呼び出しに差し替え（インターフェース `getCompatibility(wishItem, closetItems) -> CompatibilityResult` を固定しておく）
